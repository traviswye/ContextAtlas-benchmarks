## Production-tool lens (canonical framing)

The lens that made A obvious and made B converge on Reading 3 is
the production-tool framing established in ROADMAP refresh
[`f33113b`](../../contextatlas/ROADMAP.md) ("What ContextAtlas Is
FOR" subsection), preserving Travis's anchor framing:

> The benchmark repo should only be used as a tool to better our
> main repo, whose ultimate goal is to be a production tool for
> developers to use with Claude Code to enable life improvements
> for Claude (the token burn, architectural context, etc).

**A (Fix 2 default-on) under this lens:** real users running
ContextAtlas under Claude Code get cleaner bundles immediately on
default-on. Step 5 evidence has three independent strengths
(atlas-level 17→0 zero-symbol recovery, cell efficiency,
attribution clarity); Pattern 2 retention preserves rollback if a
regression surfaces post-ship.

**B (Fix 3 default-off, flag-accessible-only) under this lens:**
production target = `beta-ca` (Claude Code with CA MCP);
`alpha`/`ca`/`beta` are measurement substrates. The synthetic
ca-agent activation gap is empirically verified (Phase 6 trace
shape `{ symbol: "content" }` — no query string). But Claude
Code's production behavior under an updated tool description is
empirically unverified — qualitatively different substrate.
Shipping Fix 3 default-on would be shipping a bet on production
behavior. Reading 3 is the conservative posture that ships the
mechanism without the bet.

The lens is canonical for v0.3+ scope decisions. ADR-16 amendment
[`0bf5e2d`](../../contextatlas/docs/adr/ADR-16-bm25-symbol-context.md)
propagated the production-vs-benchmark distinction into
benchmarks-repo evidence (Step 6 evidence note caveat at
`a5808eb`). Future scope-doc authors hit the tie-break rule in
v0.3-SCOPE.md "User-facing goal" subsection ("ROADMAP wins; this
document gets amended to follow") before inheriting v0.3-SCOPE
framing.

## Evidence base (Step 5 + Step 6 spot-checks)

**Step 5 (Fix 2 — `narrow_attribution`).** Path 1 + Path 2
spot-check on p4-stream-lifecycle cell. Three independent
strengths supporting A1:
- **Atlas-level zero-symbol recovery.** Phase 6 baseline atlas
  had 17 zero-symbol claims; `drop-with-fallback` recovered all
  17 to ≥1 symbol via fallback rule.
- **Cell efficiency.** Bundle size reduced; top-INTENT precision
  improved.
- **Attribution clarity.** Per-symbol bundles now contain
  per-symbol-relevant claims rather than frontmatter-inherited
  claims that applied broadly.

Full evidence: [`v0.3-stream-a-spot-check.md`](v0.3-stream-a-spot-check.md)
§"Fix 2 spot-check on p4-stream-lifecycle" (commits `b025d3d`
main + `68e3d1e` benchmarks).

**Step 6 (Fix 3 — BM25 ranking on `get_symbol_context`).** Path 1
unit-level spot-check on Phase 6 baseline atlas + Step 5 Fix 2
atlas. Mechanism-works finding + activation-gap finding:
- **BM25 mechanism works.** All 4 probe symbols × 2 atlases =
  8/8 probe combinations show BM25-with-query produces
  measurably different (and on-topic) top-INTENT vs v0.2.
- **Cross-severity promotion fires aggressively.** 7 of 8 probe
  combinations are cross-severity promotion (BM25 promotes
  context-severity claim above hard-severity claims by query
  relevance). 1 of 8 is within-severity reordering. This is
  chain α (ADR-16 §Decision 2) doing its designed work.
- **Activation gap surfaced explicitly.** Synthetic ca-agent
  doesn't pass query parameters; production-target Claude Code's
  natural query-passing behavior under an updated tool
  description is empirically unverified.

Full evidence + Five Facts + production-vs-benchmark caveat:
[`v0.3-stream-a-spot-check.md`](v0.3-stream-a-spot-check.md)
§"Fix 3 BM25 spot-check (Step 6 append)" + §"Caveat (Step 7
dialogue insight): production-vs-benchmark distinction"

## Three-factor discipline (Decision B)

Decision B (Fix 3 ship posture) was the hardest call. Three
factors discussed independently before aggregating to a B reading
— Reading 3 as outcome of disciplined aggregation, not a priori
choice.

**Factor 1: Chain α (cross-severity promotion 7-of-8).** Chain α
fires aggressively per Step 6 evidence; production Claude Code
may trust top-INTENT more than synthetic ca-agent. Medium pull
toward Reading 3.

**Factor 2: Activation gap (synthetic vs production unverified).**
Synthetic activation gap empirically verified per ADR-16 amendment
`0bf5e2d`; Claude Code production activation under updated tool
description empirically unverified. Shipping default-on is
shipping a bet. **Load-bearing factor.**

**Factor 3: Single-query-string scope.** Step 6 tested one query
(`"response stream lifecycle read state"`); generalization
unknown. Medium pull toward Reading 3.

All three factors pull same direction; activation gap
load-bearing. For full factor-by-factor detail (pull strengths,
B1-vs-B2-vs-B3 tradeoff space, deadlock-protocol non-invocation),
see STEP-PLAN-V0.3 Step 7 progress log entry Subsection 4
[`abb18d3`](../../contextatlas/STEP-PLAN-V0.3.md).

## Pattern 2 commitment + cost honesty

Pattern 2 retention (F2 decision) means **flag retention across
both Theme 1.2 fixes** through indefinite future releases.
Rationale: first external v0.3 release wants rollback insurance
over flag-retirement cleanliness. If post-ship evidence (Stream D
+ dogfood + production user reports) shows the new defaults are
wrong, flag flip is zero-code rollback.

**Cost honesty.** Pattern 2 commits v0.3-through-v0.5+ to
maintaining flag-off codepaths until evidence-justified
retirement. The cost is bounded per-release but accumulates;
retirement deferred until Stream D evidence supports the new
default behavior. Pattern 2 maintenance carry-forward is codified
as Step 15 ship criterion (see §"Forward references" below).

## Dogfood test deferral

Bridging task between Step 7 and Step 8 — NOT a new STEP-PLAN
step. Travis runs Claude Code with CA MCP against a real repo
(NRFI / career-ops / ContextAtlas itself / OpenSCAD candidates);
~30 min, ~$0-2 spend.

**What it tests:** production-target Claude Code natural
query-passing behavior under the v0.3 `get_symbol_context` tool
description (the one that includes the optional `query` parameter
from Step 6 / ADR-16).

**What evidence it captures:** whether Claude Code's natural
query-passing behavior fires in production sessions. Empirical
resolution of the activation question that drove Reading 3's
conservative posture. Result informs v0.4 Fix 3 reconsideration
regardless of what it shows.

Reading 3 was the correct decision-under-uncertainty given what
was known at Step 7. Dogfood test resolves the uncertainty for
future decisions; it doesn't validate or invalidate Step 7's
choice in retrospect.
