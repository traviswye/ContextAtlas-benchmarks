# v0.2 Beta substrate contamination — retrospective addendum

**Status:** Filed during v0.3 Step 12 (Theme 2.1) Commit 2.
**Scope:** Research note, not a re-run. Bounded acknowledgment;
decoupled from v0.3 closure work.
**Last updated:** 2026-04-27.

## Purpose

This note documents a methodology issue identified in v0.2 Phase
5/6/7 beta-condition substrate, surfaced by the v0.3 Step 12
trace-time filter (Theme 2.1). Filed for historical hygiene; v0.2
ship is not retroactively invalidated, but the published
Beta-vs-Beta+CA narrative carries this caveat going forward.

The companion methodology note
[`atlas-file-visibility-benchmark-methodology.md`](atlas-file-visibility-benchmark-methodology.md)
covers the forward-looking decision (v0.3+ trace-time filter, v0.4
conditional clean-workspace upgrade). This retrospective covers what
the same evidence implies about v0.2's published claims.

## Finding

### What was contaminated

The trace-time filter applied to all 71 v0.2 reference cells flagged
17 (23.94%) as referencing atlas paths in tool-call args:

| Target | Phase | Total | Contaminated | Rate    |
|--------|-------|-------|--------------|---------|
| cobra  | 7     | 24    | 4            | 16.67%  |
| hono   | 5     | 23    | 7            | 30.43%  |
| httpx  | 6     | 24    | 6            | 25.00%  |
| **Overall** |  | **71** | **17**     | **23.94%** |

Per-condition concentration: **15 of 17 in `beta`** (88%); 2 of 17 in
`beta-ca` (12%; both single-evidence, likely incidental); 0 in
`alpha` or `ca`. Full per-cell evidence at
[`../scripts/v0.3-step12-backwards-apply-results.json`](../scripts/v0.3-step12-backwards-apply-results.json).

### Mechanism

The beta condition runs Claude Code CLI headless with its full
default tool set (Bash, Read, Edit, Write, Glob, Grep, etc.) and **no
MCP server declared** (per [`../RUBRIC.md`](../RUBRIC.md) §"Harness
Conditions"). When agents needed atlas content, they reached for
Bash/Read/Grep against the workspace's `atlases/<repo>/` files
(atlas.json + index.db variants).

This was structurally invited by v0.2's workspace setup: atlas
artifacts are committed to the benchmark repo (per ADR-06 in the
main ContextAtlas repository), making them visible to any agent with
file-read tools. The Beta-vs-Beta+CA comparison was designed to
isolate "MCP routing improves real CC users" but the workspace
arrangement leaked atlas substrate to beta as text files agents
could grep.

## Bias-direction analysis

Beta agents partially closed the Beta-vs-Beta+CA gap by accessing
atlas data via raw file reads instead of through the MCP routing path
the comparison intended to measure.

**Direction:** the published v0.2 Beta-vs-Beta+CA delta is an
**understatement** of the clean-workspace counterfactual. The actual
contribution of adding ContextAtlas to a real CC user's setup is
larger than v0.2 reported, because v0.2's beta numbers reflect
partial atlas access that clean-workspace beta would not have had.

**Why the bias direction matters:** in a production-tool-sharing
context (the project's anchor framing per
[`../../contextatlas/ROADMAP.md`](../../contextatlas/ROADMAP.md)
§"What ContextAtlas Is FOR"), conservative-biased claims are not
over-claims. There is no false-advertising hazard. The published
narrative ("ContextAtlas helps real Claude Code users") survives —
the reality is more favorable than the numbers indicate.

This would be different in a research-publication framing, where any
methodology hole invalidates findings regardless of bias direction.
v0.2 was not shipped under that framing.

## Why this is a research note, not a re-run

[`v0.3-SCOPE.md` Rescope Condition #4](../../contextatlas/v0.3-SCOPE.md)
explicitly authorized two paths when the >10% beta contamination
threshold triggers: (a) absorb the larger work (clean-workspace
re-run of v0.2 + v0.3 reference data) in v0.3 scope, OR (b) defer to
v0.4 with the documented methodology limit. **Path (b) was selected.**

Re-running v0.2 reference data under clean-workspace mode would cost
~$30+ in API spend (matching v0.2's own reference-run envelope) and
significant elapsed time. Path (b) trades that cost for the
methodology limit documented here and in the companion methodology
note. The trade is justified by:

- **Production-tool framing > methodology-rigor framing** (project
  anchor; [`v0.3-SCOPE.md`](../../contextatlas/v0.3-SCOPE.md)
  §"User-facing goal").
- **Conservative bias direction** (claims understated, not over-stated).
- **v0.3 thesis is unaffected** — Stream D headline findings run on
  CA/beta-ca substrate where contamination is structurally absent;
  see [`atlas-file-visibility-benchmark-methodology.md`](atlas-file-visibility-benchmark-methodology.md)
  §"Path 3b decision" for the full rationale.
- **v0.4 quality-axis measurement preserves the option** — if v0.4
  evidence shows beta contamination materially distorts findings,
  clean-workspace re-runs become warranted at that point.

## Implications for v0.2 published claims

### Affected comparisons

- **Beta-vs-Beta+CA** carries the contamination caveat. Published
  v0.2 deltas are conservative-biased; the underlying claim
  ("ContextAtlas helps real CC users") holds, but the magnitude is
  understated relative to a clean-workspace measurement.

### Unaffected comparisons

- **Alpha-vs-CA** is structurally clean (0 alpha or ca cells flagged
  across all three targets).
- **CA-vs-Beta+CA** uses two atlas-having conditions that route via
  MCP (or attempted-MCP); the 2/17 beta-ca incidental flags do not
  meaningfully shift this comparison's substrate quality.

### Implications for the v0.2 thesis

The v0.2 thesis ("works across languages and repos") is supported by
uncontaminated Alpha-vs-CA evidence across all three languages. The
Beta-vs-Beta+CA evidence that also informed the thesis carries the
caveat documented above; conservative bias direction means the
underlying claim is not over-stated, but the evidence base for
"works for real CC users" relies on the bias-direction reading
rather than clean measurement.

A clean-workspace re-run would replace the bias-direction reading
with direct measurement. That work is deferred to v0.4 as
conditional future work per Path 3b.
